    • Pros of Using JavaScript

    • Easy readability and writability.

    • Plenty of libraries and documentation to refer to.
    • First class functions and objects
    • Integratability with HTML5

    Cons of Using JavaScript

    Final Thoughts/Conclusions

    Overall, the dynamic nature of JavaScript simplified many of the difficulties associated with the implementation a Dijkstra's algorithm in the second floor of the SL Building on IUPUI's campus. Being a high level language, writability and expressiveness were two of the biggest "pros" when working with the data structures and algorithms needed for a real-world application of Dijkstra's Algorithm. Furthermore, the seamless integration of JavaScript in HTML files allowed for relative ease in using user selected information in the JavaScript code created outside of the browser environment. One of the biggest "cons" with working with JavaScript is its lack of native I/O, requiring a separate JavaScript runtime environment to be used, such as the use of Node.js in this project. Node.js allowed for the development and testing of our application outside of a browser runtime environment. However, using Node.js modules in the browser proved to be a challenge, requiring other external JavaScript tools and libraries for handling the modular programming used in Node.js applications. In summation, JavaScript worked very well for our web-based application because of its interaction with HTML documents and writability and expressiveness. While server-side development is definitely possible with tools like Node.js at our disposal, we may choose a different language for server-side aspects of a project of this nature in the future, but would still use JavaScript for client-side aspects because of the "pros" mentioned. We feel this project allowed us to better understand not only JavaScript as a programming language, but also a better understanding of when a certain langauge is most appropiate in a certain implementation.
